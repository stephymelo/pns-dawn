Overview & principles

This theme is Shopify Dawn 15.5.0 with a layer of Premier Nail Source (PNS) customizations. The goal was to replicate the look of the live premiernailsource.com site.

  • Everything custom is prefixed pns- / "PNS " — section files (sections/pns-*.liquid), CSS/JS assets (assets/pns-*.css|js), CSS classes (.pns-*), and schema/section display names ("PNS …").
  • Every section links its own CSS file. There is intentionally no shared mega-stylesheet — see architecture.
  • Two-way GitHub ↔ Shopify sync is live. Editing settings in the Shopify theme editor rewrites the JSON config files (templates/*.json, config/settings_data.json, sections/*-group.json) and pushes commits back to main. Always git pull before editing and before pushing.
  • Do not publish from git. Commits update the connected (unpublished/draft) theme in the Shopify admin; they do not publish it live.
Sync caution: because the theme editor owns the JSON, values you hard-set in *-group.json / templates/*.json can be overwritten by an editor save. For values that must survive (e.g. announcement-bar phone/email), we bake defaults into the section's Liquid or schema, not just the config.

CSS / JS architecture

Originally all custom CSS lived in one pns-custom.css (loaded globally via the header). That was split so each section owns its stylesheet.

Per-section CSS mapping

Why the card CSS is in the snippet: product cards render in many sections (collections, featured, related, search). Loading pns-card.css from card-product.liquid's style block means it loads once wherever a card appears — no section needs to remember to include it.

New custom sections

PNS Feature Bar New

Files: sections/pns-feature-bar.liquid · snippets/pns-feature-item.liquid · assets/pns-feature-bar.css

Trust-badge row (Fast Free Shipping / Customer Service / Easy Returns). Block-based: each item has an uploadable icon, heading, subtext, optional link.

  • Icon recolor: uploaded PNG/SVG icons are tinted to the "Icon color" via CSS mask (toggle "Recolor icons").
  • Editable: background color (overrides scheme), heading/accent color, icon color, bottom stroke (color + thickness), padding.
  • Mobile: the row becomes a seamless auto-scrolling marquee (duplicated set via the snippet).
  • Was originally in the header group; moved to a template section (enabled_on templates) so it's placed per-template. Currently added at the top of the homepage.

PNS Home Banner New

Files: sections/pns-home-banner.liquid · assets/pns-home-banner.css · assets/pns-home-banner.js

Two-column hero: a rotating promo banner (left) + a "Top Brands" list panel (right). Replaces the old promo-brands section.

  • Slide (banner) blocks (image + link) and Brand blocks (name + link) live in the same section.
  • Desktop layout: image locked to 808px at 2.5:1 ratio (3200×1280 artwork); Top Brands panel 390px; small black arrows sit outside the image; dots below. Mobile: full-width banner with overlaid arrows.
  • Editable NAILBUYS CTA (button color + text color pickers), rotate speed, decorative background lotus (image picker).
  • JS: <pns-home-banner> — pixel-measured translate so each slide reveals fully; pauses on hover; respects reduced motion.

PNS Featured Collection New

Files: sections/pns-featured-collection.liquid · assets/pns-featured-collection.css

A Featured-Collection variant rendered as a carousel with circular black side arrows (the shared "PNS slider" look). Copied from Dawn's featured-collection and reworked.

  • Always a slider: desktop shows 4, mobile shows ~1.5 cards (peek). Arrows hidden at the ends; "View all" removed.
  • Constrained to the theme page-width (Dawn's desktop slider is full-bleed by default — overridden).
  • Editable heading + alignment; quick_add defaults to standard so cards show the qty picker + add-to-cart.

PNS Brand Slider New

Files: sections/pns-brand-slider.liquid · assets/pns-brand-slider.css · assets/pns-brand-slider.js

Auto-rotating brand-logo carousel ("Top Brands"). Logo image blocks; rotates one logo at a time on a smooth, seamless loop; same black arrows with pink hover.

  • Editable: heading + alignment, logos-per-view (desktop), rotate interval, animation speed. Pauses on hover/focus, respects reduced motion.

PNS Footer New

Files: sections/pns-footer.liquid · assets/pns-footer.css · used via sections/footer-group.json

Custom 3-tier footer replacing Dawn's footer in the footer group.

  • Top (dark): "Menu column" blocks (link-list) + a Newsletter block (heading + description + email/Subscribe) + a decorative background lotus (image picker).
  • Contact bar (mint): address, phone, outlined-circle social icons.
  • Bottom (pink): "© {year} {brand}" (brand name in an accent color) + an editable bottom menu (link-list).
  • All three tier colors, text colors, brand color, address, phone, copyright name are editor settings. Headings/links use Dawn's footer type sizes.

PNS Breadcrumb Products New

Files: sections/pns-breadcrumb-products.liquid · assets/pns-breadcrumb-products.css

Full-width black breadcrumb bar for product pages: filled home icon + Home › collection (or fallback "All") › product title. Editable background/text colors + fallback label. enabled_on templates: [product].

PNS Related Products New

Files: sections/pns-related-products.liquid · assets/pns-related-products.css

Shopify product recommendations ("related" intent, async via <product-recommendations>) rendered in the PNS slider carousel. Heading default "You may also like". Cards use quick_add: standard. Product template only.

PNS Related Collections New

Files: sections/pns-related-collections.liquid · assets/pns-related-collections.css

Same slider, but products are grouped by the product_collection.product_collection metafield (not the Shopify collection) — it loops the product's collections, de-dupes, and shows others sharing the same metafield value, excluding the current product. Heading default "More From This Collection".

Dependency: requires the product_collection.product_collection product metafield to be populated, or the section renders nothing.

PNS Brands Page New

Files: sections/pns-brands-page.liquid · used via templates/page.shop-by-brand.json

"Shop by Brand" grid looping shop.vendors, linking each to /collections/<vendor-handle>, greyscale→color on hover, responsive 4→3→2→1. Editable heading + description (no blocks). Styles are inline in the section.

Theme-editor setup checklist

What has to be configured in the Shopify theme editor for the theme to look complete.

  • Homepage: add PNS Feature Bar (top), PNS Home Banner, PNS Featured Collection(s), PNS Brand Slider, multicolumns.
  • Product template: add PNS Breadcrumb Products (top), PNS Related Products and/or PNS Related Collections (replace/hide Dawn's "Related products").
  • Shop-by-Brand page: template page.shop-by-brand.json uses PNS Brands Page.
  • Footer: footer group uses PNS Footer — set the two menu columns (footer / footer-products menus), address, phone, colors, and upload the background lotus.
  • Upload assets: footer lotus logo; Feature Bar icons (shipping / customer-service / return); brand logos (Brand Slider + Top Brands); banner artwork (3200×1280).
  • Social links in Theme settings → Social media (Facebook / Instagram / YouTube / TikTok) — used by the announcement bar and footer.
  • Header: set the main menu (drives the secondary nav bar) and the nav_link_color if changing from #4A4A4A.
  • Logo: Theme settings → logo width (max raised to 500px). Use a source image ≥1000px wide to stay crisp.
  • PNS Home Banner: set the NAILBUYS button link + colors; add banner slides + Top Brands entries.
  • Metafield: populate product_collection.product_collection on products if using PNS Related Collections.

Rimix product badges — how it works & why it can look broken

Verified 2026-07-16 by inspecting the live public storefront (premiernailsource.com) and Rimix's own script. None of this lives in the theme — it's a site-wide app embed configured in the Rimix dashboard.

app-embed What Rimix is in this store

  • Single site-wide embed: shopify://apps/rimix-product-badges/blocks/rimix-helper (in config/settings_data.json, disabled:false, empty settings). There is no Rimix app block in any template or section.
  • All logic lives in the app's script product-badges.js (loaded from the Shopify app CDN). The theme contributes nothing except having card markup the script can match.

The mechanism (this is the important part)

The script has zero hardcoded theme selectors. It reads a per-theme selector profile from the global window.RMPB_COMPATIBLE_THEME.selectors. For this store (theme "Dawn", id 887) the collection/card (loop) selectors are:

loop.wrapper : .card-wrapper , .spf-product-card
loop.image   : .card__media , .spf-product-card__image-wrapper
loop.title   : .card > .card__content .card__heading , .spf-product-card__title
loop.price   : .card > .card__content .price , .spf-product-card__price-wrapper

Per card it: finds the a[href*="/products/"] link → .closest(loop.wrapper) → injects mount points <div class="rm-product-badge-wrap" data-position="inside_image|above_title|below_title|above_price|below_price"> relative to image/title/price → fills the ones whose product matches a badge condition.

The .spf-product-card* half of each selector was hand-added in the Rimix dashboard to support the Smart Product Filter (SPF) app, which replaces the grid on collection pages. On /collections/all there are 0 Dawn cards — 100% spf-product-card. Home page = native Dawn .card-wrapper cards.

Our pns-dawn card already satisfies every hook ✅

Tracing snippets/card-product.liquid, the Dawn 15.5 card exposes all four loop selectors:

  • .card-wrapper ✅ · .card__media ✅
  • .card > .card__content .card__heading ✅ — the second .card__content is a direct child of .card.
  • .card > .card__content .price ✅ — from the price snippet render.

The Dawn 15.5 <product-component> wrapper sits above .card-wrapper, so it does not break .closest() or any descendant query. And pns-card.css hides only .card__badge > .badge (Dawn's native span) — never Rimix's .rm-product-badge-wrap. So the card markup is not the blocker.

If badges don't show on the new theme, it's one of these (all app-side)

CauseFix (Rimix dashboard — not theme code)
Theme profile not applied. The selector table is stored per-theme; a new/previewed theme may load no selectors.Re-run Rimix's theme setup for the new theme, or paste the custom selectors: wrapper .card-wrapper, image .card__media, title .card > .card__content .card__heading, price .card > .card__content .price.
No matching condition. On the live home page Rimix injected a mount point into 68/68 cards but only 14 showed a badge — badges are gated on tags (e.g. special offer) + inventory (≤15).Test with a product that actually meets a badge's condition, or loosen the condition.
Dynamic cards. Rimix scans on load and marks links data-rmpb="loaded". Cards rendered after that — slick-slider clones, async PNS Related Products — can be skipped.App-side (Rimix's mutation watch) — be aware async sections are the risky ones.
Hard rule: never open the live PNS theme editor / Shopify admin. This was all diagnosed read-only from the public storefront + the app's public script.

Gotchas & conventions for the next dev

  • Always git pull before editing and before pushing — the Shopify editor pushes its own commits.
  • New section? Prefix it pns- / "PNS ", give it its own assets/pns-*.css, and link that CSS from the section's first line.
  • Card CSS is loaded by snippets/card-product.liquid, not by each section — don't re-add it per section.
  • Slider sections reuse Dawn's <slider-component>: slides need id="Slide-{{ section.id }}-…" and the grid needs id="Slider-{{ section.id }}" or the arrows won't scroll.
  • Values that must persist (phone/email, etc.) belong in Liquid default filters or schema defaults, not only in the editor-owned JSON.
  • Don't publish from git — commits only update the draft theme.
  • Base theme is Dawn 15.5.0. A future Dawn upgrade will need these customizations re-applied to the new Dawn files (they were re-applied once already, 15.4.1 → 15.5.0).
